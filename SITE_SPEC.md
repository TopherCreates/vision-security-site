# Vision Security Website Build Specification

**Version:** 1.0
**Date:** 2026-05-26
**Sources of truth:** Vision Security Brand Bible (May 2026), current visionsecurity.net audit, industry best practices for life-safety and alarm-system websites.
**Marketing automation platform:** Scale (white-labeled GoHighLevel / HighLevel).

## 1. Executive Summary

This document specifies the full website build for Vision Security in three phases:

| Phase | Pages | Purpose | Recommended timeline |
|---|---|---|---|
| Phase 1 (MVP) | 37 pages | Launchable site, all primary funnels live, all SEO-critical pages indexed | Months 1 to 3 |
| Phase 2 (Growth) | +8 pages | Content marketing engine, careers, deeper trust signals | Months 4 to 6 |
| Phase 3 (Scale) | +6 pages | Case studies, project gallery, additional lead magnets, press | Months 7 to 12 |

Three already exist in the mockup (Home, One Vision Program, Property Managers). The remaining 34 MVP pages need to be built, then 14 additional pages across phases 2 and 3.

The site integrates with Scale (GoHighLevel) for every form, chat, phone, SMS, and email touchpoint. Scale becomes the single source of truth for every lead, client communication, and follow-up workflow.

## 4. Forms Inventory

Every form posts to Scale (GoHighLevel) via a dedicated webhook or embedded form. Lead source tag is set per form. Auto-response (email + SMS) fires within 60 seconds. Internal alert routes to the right specialist.

### 4.1 Form catalog

| # | Form | Pages it appears on | Lead source tag | Routing in Scale |
|---|---|---|---|---|
| 1 | Request Onsite Assessment | Home, all service pages, all geo pages | `assessment_request` | New Lead pipeline → Residential or Commercial stage based on form field |
| 2 | Request Compliance Review | One Vision, Property Managers, Hospitality, Multi-Family, CRE | `compliance_review` | Commercial pipeline → Compliance Review stage |
| 3 | Schedule Service | Service pages, Contact, footer | `service_request` | Service Operations pipeline |
| 4 | Billing Question | Contact, footer | `billing` | Admin / Billing queue |
| 5 | Request a Callback | All CTA strips, mobile sticky bar | `callback` | New Lead pipeline → Quick Call stage; SMS to designated specialist |
| 6 | Compliance Calendar Download | Compliance Calendar landing page, sidebar offers | `lead_magnet_calendar` | Nurture pipeline → Educational sequence |
| 7 | Newsletter Signup | Footer, blog pages | `newsletter` | Marketing list |
| 8 | Career Application | Careers page | `careers` | HR queue; resume upload to GHL document storage |
| 9 | Contact (General) | Contact page fallback | `general` | New Lead pipeline → Triage stage |
| 10 | Chat handoff | Chat widget fallback when unattended | `chat_handoff` | New Lead pipeline → Live Chat stage |

### 4.2 Field specifications

**Standard fields (every form):**
- First Name (required)
- Last Name (required)
- Email (required, validated)
- Phone (required, validated for US format)
- Honeypot (anti-spam, hidden)
- reCAPTCHA v3 (invisible)
- SMS consent checkbox (TCPA / 10DLC compliance, defaults unchecked)
- Marketing consent checkbox (CAN-SPAM compliance, defaults unchecked)

**Form-specific additions:**

| Form | Additional fields |
|---|---|
| Request Onsite Assessment | Property Type (radio: Residential / Commercial), Property Address, Service Interest (multi-select: Fire / Security / Video / Access / Inspection), Best Time to Reach, Notes |
| Compliance Review | Company Name, Number of Properties, Property Types, Current Inspection Status, Decision Timeframe |
| Schedule Service | Existing Client Account # (optional), Property Address, Service Type, Urgency (Emergency / Same-day / Within a week), Description |
| Billing Question | Account # or Invoice #, Type of Question (dropdown), Description |
| Compliance Calendar Download | Role (dropdown: Property Manager / Owner / Builder / Other), Portfolio Size |
| Career Application | Role applying for, Resume upload (PDF or DOCX), LinkedIn URL, Cover letter |

### 4.3 Confirmation behavior

Every form submission triggers:

1. Immediate on-page confirmation message ("Thanks. A specialist will reach you within one business hour.")
2. Auto-response email from Scale (branded, addressed by name, includes next steps)
3. Auto-response SMS (only if SMS consent given): "Vision Security received your request. We'll be in touch shortly. Reply STOP to opt out."
4. Internal alert: SMS + email to the routed specialist
5. Scale pipeline record created with all metadata

For emergency or same-day service requests, a separate hot-lead workflow:
1. Phone call to dispatcher within 5 minutes during business hours
2. Pager / on-call alert outside business hours

## 5. Scale (GoHighLevel) Integration

### 5.1 Communication touchpoints

| Touchpoint | Integration method | Notes |
|---|---|---|
| Forms | GHL embedded form OR custom form POSTing to GHL Forms API or webhook | Custom forms recommended for design control; GHL receives via webhook |
| Chat | GHL Web Chat widget on every page | Configure office hours, away message, lead handoff, conversation transfer to SMS |
| Phone | GHL tracked numbers (dynamic number insertion per campaign / page) | Forwards to main line; call records logged to lead profile |
| SMS (2-way) | GHL 10DLC registered number | All marketing SMS opt-in; transactional SMS for appointment reminders, post-service follow-up |
| Email | GHL workflows for nurture sequences; transactional via GHL SMTP | Branded templates; deliverability optimized via SPF, DKIM, DMARC setup |
| Calendar booking | GHL appointment booking widget on Contact, Services, Persona pages | Optional but high-conversion for "book consultation" CTAs |
| Reviews | GHL review request automation post-service | Auto-asks for Google review 7-10 days after job complete (Brand Bible Priority 2 target: 10+ new reviews/month) |

### 5.2 Pipelines

Two distinct pipelines in Scale:

**Residential B2C pipeline:**
- New Lead → Quick Call → Onsite Scheduled → Onsite Complete → Proposal Sent → Won / Lost
- Tags: Estate, Multi-Property, Local Homeowner

**Commercial B2B pipeline:**
- New Lead → Compliance Review Scheduled → Property Walked → Proposal Sent → Negotiation → Won / Lost
- Tags: Property Manager, GC, Hospitality, Multi-Family, CRE, Construction

Both pipelines have automated nurture for stale leads (no response in 7, 14, 30 days).

### 5.3 Tracking and attribution

- UTM parameters captured on every form
- Tracked phone numbers per market and per persona page (Aspen has its own DNI number, Property Managers page has its own, etc.)
- Google Ads, Facebook Pixel, and GHL conversion tracking unified
- GHL reports configured for: lead source by market, lead source by service, conversion rate per persona page, average time to first contact

### 5.4 Workflows to build in Scale

| Workflow | Trigger | Actions |
|---|---|---|
| New Lead Welcome | Any form submission | Auto-email + auto-SMS + internal alert + assign owner |
| Hot Lead (same-day service) | "Urgency: Emergency" or "Same-day" selected | Phone alert to dispatcher; SMS to on-call tech; escalation if no acknowledgement in 5 min |
| Compliance Calendar Nurture | Calendar download | 5-email sequence over 30 days; final email offers Compliance Review |
| Onsite Reminder | 24 hours before scheduled onsite | SMS + email to client; SMS to tech |
| Post-Service Follow-Up | 2 weeks after invoice marked paid | SMS to client + Google review request |
| Stale Lead Re-Engagement | 14 days no activity | Email + SMS with new educational content |
| Annual Inspection Reminder | 11 months after last inspection | Email + SMS + dispatcher task |
| Birthday / Holiday | Date-based | Optional brand-warmth touchpoint |

### 5.5 Compliance considerations

- **10DLC SMS registration** required (A2P 10DLC). Allow 4 to 6 weeks for carrier approval.
- **TCPA compliance:** every SMS opt-in must be explicit, double-confirmed, with documented timestamp and IP.
- **CAN-SPAM:** every marketing email needs physical address and one-click unsubscribe.
- **CCPA / Colorado privacy laws:** add Do Not Sell My Info link, support data deletion requests via Scale.
- **HIPAA:** not applicable unless healthcare facilities are served and client data involves protected health info. Unlikely for life-safety category but flag for legal review.

## 7. SEO Strategy

### 7.1 Geographic SEO

Each market lander targets:
- City + service combinations: "fire alarm inspection Aspen CO," "security system installation Glenwood Springs"
- City + audience combinations: "property manager security Vail," "estate fire alarm Snowmass"
- Local pack signals: NAP consistency (Google Business Profile, Yelp, BBB, Nextdoor)
- Schema markup: `LocalBusiness`, `Service`, `Place`, `GeoCoordinates`

### 7.2 Vertical SEO

Each vertical lander targets:
- Industry-specific terms: "resort fire alarm system Colorado," "multi-family access control Aspen"
- Pain-point keywords: "multi-property compliance audit," "hotel false alarm prevention"
- Schema: `Service`, `Article` for any embedded education

### 7.3 Site-wide SEO

- `Organization` and `LocalBusiness` schema on every page
- `BreadcrumbList` schema on nested pages
- `FAQPage` schema on FAQ and on any page with FAQ accordions
- `Service` schema on each service page
- `Review` schema (when 5+ verified reviews are surfaced inline)
- Open Graph and Twitter Card meta on every page
- XML sitemap auto-generated and submitted to Google Search Console + Bing Webmaster
- robots.txt configured to allow indexing
- Canonical tags on every page
- 301 redirect map from every current visionsecurity.net URL to the new equivalent (see Section 9.4)

### 7.4 Content cadence (Phase 2)

Blog publishing schedule: 2 articles per month minimum, evergreen and topical mix.

Target topics (Brand Bible Priority 5):
- NFPA code changes affecting Roaring Fork Valley properties
- How to evaluate a life safety vendor
- Real cost of missed compliance deadlines
- Annual inspection checklist by property type
- Fire marshal inspection prep
- Security system retrofit considerations for historic properties (Aspen, Carbondale)

## 8. Technical Specifications

### 8.1 Stack recommendation

Per project direction (off Thryv), recommend a modern static-site framework + headless CMS architecture:

| Layer | Recommendation | Rationale |
|---|---|---|
| Frontend | Astro or Next.js (static export) | Fast, SEO-friendly, low hosting cost, easy vendor handoff |
| CMS | Sanity, Contentful, or Storyblok | Non-technical content editing; Sanity preferred for flexibility |
| Hosting | Vercel, Netlify, or Cloudflare Pages | Edge CDN, automatic deploys, built-in analytics |
| Forms | Custom forms POSTing to Scale webhook | Design control + Scale captures the lead |
| Analytics | Google Analytics 4 + Google Tag Manager + Scale's built-in analytics | GA4 for traffic, GTM for tag management, Scale for lead and conversion |
| Search Console | Google Search Console + Bing Webmaster | SEO monitoring |

Alternative if vendor prefers WordPress:
- WordPress with custom theme built from these mockups
- Gravity Forms or Fluent Forms for form handling (webhook to Scale)
- Yoast or RankMath for SEO
- Higher maintenance overhead, more familiar to many vendors

### 8.2 Performance targets

- Lighthouse Performance score: 90 or higher
- Largest Contentful Paint (LCP): under 2.5 seconds
- First Input Delay (FID): under 100 milliseconds
- Cumulative Layout Shift (CLS): under 0.1
- Mobile-first: every page tested at 360 px width, 768 px tablet, 1280+ desktop
- Hero video: 1080p with 720p mobile fallback, lazy-loaded, autoplays muted
- Image strategy: WebP with JPG fallback, lazy-loaded below the fold, responsive `srcset`

### 8.3 Accessibility

WCAG 2.1 Level AA compliance:

- Color contrast verified for all text on all backgrounds
- All images have descriptive alt text (already in mockups)
- Keyboard navigation supported throughout (tab order, focus styles)
- Skip-to-content link in nav
- Form labels associated with inputs; error messages announced to screen readers
- Video has captions or descriptive transcripts
- `prefers-reduced-motion` respected (already implemented in marquee)
- Click targets minimum 44 px on mobile

### 8.4 Security and compliance

- HTTPS (TLS 1.3) on all pages
- HSTS header
- Content Security Policy (CSP) configured
- Cookie consent banner (GDPR-style) for analytics and marketing cookies
- Privacy policy and Terms accessible from every page footer
- DocuSign or similar e-signature for service agreements (post-conversion, not on site)

### 8.5 Browser support

- Last two versions of Chrome, Firefox, Safari, Edge
- iOS Safari 14+
- Android Chrome current
